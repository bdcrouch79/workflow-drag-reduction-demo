# Workflow Rules

## Routing Logic

- `Document Support` requests route first to Lena Brooks or Noah Whitaker depending on project documentation needs.
- `Project Setup Change` requests route to Operations Support for validation, then to the relevant project manager for confirmation.
- `Closeout Assistance` requests route to Lena Brooks, Monica Reeves, or the assigned project manager depending on whether the need is documentation, financial, or turnover related.
- `Reporting Request` items route to Kelsey Morgan or Evan Cross.
- `Field Coordination` requests route to Caleb Dyer, Jordan Hayes, or a superintendent based on field ownership.
- `Vendor Setup` requests route to Monica Reeves with controller approval when financial setup is required.
- `Operations Support` requests route to Olivia Chen or Rachel Holloway depending on scope.

## Approval Logic

- low-risk requests can move directly from triage to assignment
- requests affecting cost, vendor onboarding, staffing, or client commitments require approval
- critical and some high requests may skip nonessential stages but still record final approval
- rejected requests remain visible for audit and trend review
